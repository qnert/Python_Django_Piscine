Ex01: Display process of a static page.

{% block style %} {% endblock style %}
  1. Request Handling:
    • When a user's browser sends a request for a particular URL, it's intercepted by Django's URL dispatcher.
    • The URL dispatcher maps the requested URL to a specific view function based on the URL patterns defined in the project's urls.py file.
  2. View Function Execution:
    • Once the URL dispatcher determines the appropriate view function, it invokes that function.
    • The view function performs any necessary business logic, such as fetching data from a database or processing user input.
    • In the case of displaying a static web page from a simple template, the view function typically renders a template using Django's template engine.
  3. Template Rendering:
    • The view function renders a template by combining it with context data.
    • Context data may include variables or objects that are passed from the view to the template, allowing dynamic content to be displayed.
  4. HTML Response Generation:
    • Once the template is rendered with the appropriate context data, Django generates an HTML response.
    • This HTML response includes the content of the rendered template, which may include static HTML markup as well as dynamically generated content.
  5. Sending Response to Browser:
    • Django sends the HTML response back to the user's browser as an HTTP response.
    • The browser receives the response and renders the HTML content, displaying the static web page to the user.
